---

🟠 1. Polar Configuration (Spherical Robot)

What it is:

This robot has three rotating joints that let the end of the arm (called the end effector) move in a sphere-like area.


How it moves:

One joint rotates around the vertical axis (turns left or right).

Another rotates around the horizontal axis (moves up or down).

The third joint lets the end effector spin.


Where it's used:

For tasks needing high precision, like assembling electronics or delicate surgery.

Also used in VR, computer graphics, etc.


Disadvantages:

Can’t move very high vertically.

Complex to design and control.



---

🔵 2. Cylindrical Configuration

What it is:

This robot has one rotating joint and one sliding (linear) joint that lets the arm move like a cylinder shape.


How it moves:

It rotates around a vertical axis.

It slides up and down, or in and out in a straight line.


Where it's used:

Common in welding, drilling, or tasks that need the arm to move in a straight line while keeping a fixed distance from the object.


Advantages:

Good for large work areas and simple, repetitive tasks.



---

🟢 3. Cartesian Configuration

What it is:

This robot moves along three straight lines (X, Y, Z axes) – like moving left-right, forward-backward, and up-down.


How it moves:

It moves in straight lines only, not by rotating.


Where it's used:

Great for pick and place operations, 3D printing, CNC machines, and assembly lines.


Advantages:

Very precise and simple to control.

Good for box-shaped work areas.


Disadvantages:

Not very flexible.

Can’t reach around or in tight spaces like other robots.



---

Robot Type	Movement Shape	Good For	Limitations

Cylindrical	Cylinder-like	Welding, drilling	Less flexible

Saves data permanently.

Includes hard disks, SSDs, pen drives, etc.

from passlib.context import CryptContext
from datetime import datetime, timedelta
from jose import jwt, JWTError
from fastapi import HTTPException, Depends
from fastapi.security import OAuth2PasswordBearer
from app.config import settings
from app import crud
from sqlalchemy.orm import Session
from app.database import SessionLocal

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/token")

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def verify_password(plain_password: str, hashed: str) -> bool:
    return pwd_context.verify(plain_password, hashed)

def create_access_token(data: dict, expires_delta: timedelta | None = None):
    to_encode = data.copy()
    expire = datetime.utcnow() + (expires_delta or timedelta(minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES))
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, settings.SECRET_KEY, algorithm=settings.ALGORITHM)

def get_current_user(token: str = Depends(oauth2_scheme), db: Session = Depends(get_db)):
    credentials_exception = HTTPException(status_code=401, detail="Could not validate credentials")
    try:
        payload = jwt.decode(token, settings.SECRET_KEY, algorithms=[settings.ALGORITHM])
        user_id: int = payload.get("sub")
        if user_id is None:
            raise credentials_exception
    except JWTError:
        raise credentials_exception
    user = crud.get_user_by_id(db, user_id)
    if not user:
        raise credentials_exception
    return user

from sqlalchemy.orm import Session
from app import models, schemas
from datetime import date

def get_user_by_id(db: Session, user_id: int):
    return db.query(models.User).filter(models.User.id == user_id).first()

def get_user_by_username(db: Session, username: str):
    return db.query(models.User).filter(models.User.username == username).first()

def get_user_by_email(db: Session, email: str):
    return db.query(models.User).filter(models.User.email == email).first()

def create_user(db: Session, user: schemas.UserCreate, hashed_password: str):
    db_user = models.User(
        username=user.username, email=user.email, phone=user.phone, hashed_password=hashed_password
    )
    db.add(db_user)
    db.commit()
    db.refresh(db_user)
    return db_user

def create_task(db: Session, user_id: int, task_in: schemas.TaskCreate):
    db_task = models.Task(
        title=task_in.title, description=task_in.description, date=task_in.date, owner_id=user_id
    )
    db.add(db_task)
    db.commit()
    db.refresh(db_task)
    return db_task

def get_tasks_for_user(db: Session, user_id: int):
    return db.query(models.Task).filter(models.Task.owner_id == user_id).all()

def get_tasks_for_user_by_date(db: Session, user_id: int, on_date: date):
    return db.query(models.Task).filter(models.Task.owner_id == user_id, models.Task.date == on_date).all()

def get_task(db: Session, user_id: int, task_id: int):
    return db.query(models.Task).filter(models.Task.owner_id == user_id, models.Task.id == task_id).first()

def delete_task(db: Session, user_id: int, task_id: int):
    task = get_task(db, user_id, task_id)
    if task:
        db.delete(task)
        db.commit()
        return True
    return False

def delete_all_tasks(db: Session, user_id: int):
    db.query(models.Task).filter(models.Task.owner_id == user_id).delete()
    db.commit()
    return True

def update_user_settings(db: Session, user_id: int, notifications_enabled: bool = None, is_24h_format: bool = None):
    user = get_user_by_id(db, user_id)
    if not user:
        return None
    if notifications_enabled is not None:
        user.notifications_enabled = notifications_enabled
    if is_24h_format is not None:
        user.is_24h_format = is_24h_format
    db.add(user)
    db.commit()
    db.refresh(user)
    return user

def change_user_password(db: Session, user_id: int, new_hashed: str):
    user = get_user_by_id(db, user_id)
    if not user:
        return None
    user.hashed_password = new_hashed
    db.add(user)
    db.commit()
    db.refresh(user)
    return user

@app.post("/auth/token", response_model=schemas.Token)
def login(form_data: OAuth2PasswordRequestForm = Depends(), db: Session = Depends(get_db)):
    user = crud.get_user_by_username(db, form_data.username) or crud.get_user_by_email(db, form_data.username)
    if not user or not auth.verify_password(form_data.password, user.hashed_password):
        raise HTTPException(status_code=401, detail="Incorrect credentials")
    token = auth.create_access_token({"sub": user.id})
    return {"access_token": token, "token_type": "bearer"}

@app.post("/auth/signup", response_model=schemas.UserOut)
def signup(user_in: schemas.UserCreate, db: Session = Depends(get_db)):
    if crud.get_user_by_username(db, user_in.username):
        raise HTTPException(400, "Username already taken")
    if crud.get_user_by_email(db, user_in.email):
        raise HTTPException(400, "Email already registered")
    hashed = auth.hash_password(user_in.password)
    return crud.create_user(db, user_in, hashed)

    if update.username:
        if crud.get_user_by_username(db, update.username) and update.username != user.username:

        user.username = update.username
    if update.email:

            raise HTTPException(400, "email taken")
        user.email = update.email
    if update.phone is not None:
        user.phone = update.phone
    db.add(user)
    db.commit()
    db.refresh(user)
    return user

Java Buzzwords

When Java was introduced by Sun Microsystems, it came with a set of keywords or features called buzzwords that made Java unique and popular. These buzzwords describe the main design goals and characteristics of the Java language.

List of Java Buzzwords

(Java is often described with these 12 buzzwords)

Simple

Java syntax is easy to learn, similar to C/C++ but simplified (no pointers, operator overloading, multiple inheritance).

Object-Oriented

Everything in Java is treated as an object.

Supports concepts: Encapsulation, Inheritance, Polymorphism, Abstraction.

Distributed

Java provides libraries for networking, RMI (Remote Method Invocation), web technologies — useful for distributed applications.

Robust

Strong memory management, exception handling, garbage collection → prevents crashes.

Secure

Java programs run inside JVM (sandbox environment).

No direct memory access like pointers → safer against viruses/hacks.

Architecture-Neutral

Java bytecode can run on any machine with JVM, independent of CPU or OS.

Portable

Same code runs anywhere without modification.

Data types have fixed sizes (e.g., int = 32 bits everywhere).

Interpreted

Java source code → compiled into bytecode → executed by JVM interpreter.

High Performance

JIT (Just-In-Time) compiler improves execution speed, making Java faster than traditional interpreted languages.

Multithreaded

Java supports multiple threads of execution.

Useful for animations, games, multimedia, concurrent tasks.

Dynamic

Java supports runtime polymorphism, dynamic class loading, and reflection.

Distributed (sometimes grouped separately from networked)

Enables easy creation of networked/distributed apps using APIs like RMI, CORBA, sockets.

21/11/2025, 1:14 pm - Sammu: What is an Addressing Mode?
Addressing mode is the method a CPU uses to locate the operand (data) needed for an instruction.
In simple terms: It tells the processor where to find the data — inside a register, in memory, as a constant, or somewhere relative to another address.

Addressing modes give flexibility to assembly instructions and help reduce program size and complexity.

Different Types of Addressing Modes (with examples)
Below are the main modes you’re expected to write for exams.

1. Immediate Addressing Mode
The operand is given directly in the instruction.

Example:

nginx
Copy code
MOV R1, #10
Here 10 is directly available — no memory lookup.
Fastest, but works only for constants.

2. Register Addressing Mode
The operand is inside a CPU register.

Example:

nginx
Copy code
MOV R2, R5
The instruction copies the value from register R5 to R2.
No memory access → very fast.

3. Direct Addressing Mode
The instruction contains the memory address of the operand.

Example:

yaml
Copy code
MOV R1, 5000
This means: load into R1 the value stored at memory address 5000.

Simple, but limited by address size.

4. Indirect Addressing Mode
The instruction contains an address of a register, and that register contains the actual memory address of the data.

Example:

scss
Copy code
MOV R1, (R3)
If R3 = 8000, then data is taken from memory location 8000.

Useful for pointers.

5. Indexed Addressing Mode
A base address is combined with an index (usually in a register) to get the final address.

Example:

scss
Copy code
MOV R1, 2000(R5)
Final address = 2000 + value in R5.
Often used for arrays.
21/11/2025, 1:17 pm - Sammu: Byte Order

Big Endian: Stores the most significant byte (MSB) at the lowest memory address.

Little Endian: Stores the least significant byte (LSB) at the lowest memory address.

Memory Representation Example (0x12345678)

Big Endian: Stored as 12 34 56 78.

Little Endian: Stored as 78 56 34 12.

Human Readability

Big Endian: Matches the natural way humans write numbers (from most significant to least significant).

Little Endian: Appears reversed to human readers.

Processor Usage

Big Endian: Used by some RISC processors and network protocols.

Little Endian: Used by most modern processors like Intel and AMD.

Data Transfer Compatibility

Big Endian: Preferred for network communication (network byte order).

Little Endian: Needs conversion when communicating over networks to avoid misinterpretation.

Virtual Machine – Concept with Example

Concept of Virtual Machine

A Virtual Machine (VM) is a software-based abstraction of physical hardware such as CPU, memory, disk drives, and network devices.
It creates the illusion of multiple independent computers running on a single physical machine.
Each virtual machine behaves as if it has its own processor and memory, even though resources are shared.


---

Host OS and Guest OS

The main operating system installed on the hardware is called the Host Operating System.

The operating systems running inside virtual machines are called Guest Operating Systems.

Multiple guest OSs can run simultaneously on the same host system.



---

Implementation of Virtual Machine

The virtual machine software runs in kernel mode as it controls hardware resources.

Guest operating systems run in user mode.

The VM software provides an exact duplicate of the underlying hardware to the guest OS.

This separation ensures safe execution and isolation.



---

Benefits of Virtual Machines

Multiple operating systems can share the same hardware efficiently.

Isolation improves security; a virus in one guest OS does not affect others or the host.

Software resources can be shared using:

Shared file systems

Virtual communication networks


Enables system development and testing without stopping the main system.

Supports system consolidation, reducing hardware and maintenance costs.



---

Example – VMware

VMware is a popular virtualization software.

It runs as an application on a host OS such as Windows or Linux.

It allows multiple guest operating systems like Linux, Windows XP, FreeBSD to run concurrently.

Each VM has its own virtual CPU, memory, disk, and network interface.

The virtualization layer abstracts physical hardware into isolated virtual machines.

Explain ARM processor pipeline with suitable example.

**Answer**
The ARM processor uses pipelining to improve performance by overlapping the execution of multiple instructions. Pipelining allows different stages of instruction processing to be carried out simultaneously, thereby increasing instruction throughput.

In a basic ARM processor, a **3-stage pipeline** is used. The stages are
• Fetch
• Decode
• Execute

**Fetch stage**
The instruction is fetched from memory using the program counter (PC).

**Decode stage**
The fetched instruction is decoded, and the required operands are read from registers.

**Execute stage**
The instruction is executed, and the result is written back to the destination register.

While one instruction is being executed, the next instruction is decoded, and another instruction is fetched. This overlapping of operations increases speed.

**Example illustrating pipeline operation**

Instruction 1: ADD R0, R1, R2
Instruction 2: SUB R3, R4, R5
Instruction 3: MOV R6, R7

Fetch ADD

Clock cycle 2
Decode ADD
Fetch SUB

Clock cycle 3
Execute ADD
Decode SUB
Fetch MOV

Clock cycle 4
Execute SUB
Decode MOV

Clock cycle 5
Execute MOV

Thus, instead of completing one instruction fully before starting the next, the ARM pipeline allows multiple instructions to be processed at the same time, improving overall performance.
